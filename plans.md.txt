# DONEXT MVP — Full Build Prompt

You are building a web application called **DoNext** — an AI-powered life planner that solves decision paralysis. The core problem: people feel stuck not because they're lazy, but because they have too many things they could do and can't decide what to do right now. DoNext takes their vague weekly goals, combines them with their real-life schedule, and uses AI to generate a concrete daily task plan. During the day, the app shows only ONE task at a time — eliminating the decision-making burden entirely.

---

## TECH STACK

- **Frontend:** React 18 + Vite
- **Styling:** Tailwind CSS (dark mode only for MVP)
- **Charts:** Recharts (for weekly score visualization)
- **Backend & Database:** Supabase (PostgreSQL + Auth + Row Level Security)
- **Auth:** Supabase Auth (email/password + Google OAuth)
- **AI:** Anthropic Claude API (called from a Supabase Edge Function to keep the API key server-side)
- **Hosting:** Vercel (frontend), Supabase (backend)
- **Routing:** React Router v6
- **State Management:** React Context + hooks (no Redux)
- **Date handling:** date-fns library

## SUPABASE EDGE FUNCTION — AI Task Generation

Create a Supabase Edge Function at `supabase/functions/generate-tasks/index.ts`. This keeps the Anthropic API key server-side and secure.

The function receives the user's fixed schedule, sleep times, available hours, weekly goals per pillar, and pillar info. It calls Claude to decompose goals into scheduled tasks.

```typescript
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";

const ANTHROPIC_API_KEY = Deno.env.get("ANTHROPIC_API_KEY");

const SYSTEM_PROMPT = `You are DoNext's AI task planning engine. Your job is to take a user's weekly goals across 5 life pillars and generate a concrete, day-by-day schedule of tasks that fits within their available free time.

You will receive:
- The user's fixed commitments (work, school, commute, etc.) per day
- Their sleep schedule
- Their available free hours per day (already calculated — trust these numbers)
- Their weekly goals for each pillar (in natural language)
- The pillar names, icons, and IDs

YOUR RULES:
1. Only schedule tasks within the available hours for each day. NEVER exceed available time.
2. Schedule only 80% of available time — leave 20% as buffer for overflow and rest.
3. Every task MUST have: a clear title (action-oriented verb + specific outcome), a duration (only 15, 30, 45, 60, or 90 minutes), and a completion_criteria field (one sentence: "Done when...").
4. Place cognitively demanding tasks (studying, creative work, deep focus) earlier in the day when possible. Place physical tasks (gym, walks) in late afternoon/evening. Place social tasks (calls, hangouts) in evenings or weekends.
5. Vary tasks across days. Do NOT repeat the same task every day. Break goals into different sub-tasks for different days.
6. Respect dependencies: foundational tasks before advanced ones (e.g., "research" before "write", "design" before "build").
7. Distribute tasks across pillars so no single pillar dominates any given day. Aim for 2-3 different pillars touched per day.
8. If a goal is too ambitious for the available time, still create tasks but note in a "warnings" field that the goal may not be fully achievable.
9. Include at least one short (15-30 min) easy task at the start of each day's scheduled tasks. This builds momentum.
10. For each task, assign an energy_level: "high" for creative/deep work, "medium" for moderate focus, "low" for routine/easy tasks.
11. Set sort_order as integers starting from 1 for each day, indicating the recommended execution order.
12. Set scheduled_time as the suggested start time in HH:MM format (24-hour).

RESPOND WITH ONLY valid JSON in this exact format, nothing else:
{
  "tasks": [
    {
      "pillar_id": "<uuid of the pillar>",
      "day": "YYYY-MM-DD",
      "scheduled_time": "HH:MM",
      "title": "Action-oriented task title",
      "completion_criteria": "Done when...",
      "duration_minutes": 30,
      "energy_level": "high",
      "sort_order": 1
    }
  ],
  "warnings": ["Optional array of warnings if goals seem too ambitious"],
  "summary": "A brief 2-3 sentence summary of the week plan for the user"
}`;

serve(async (req) => {
  // Handle CORS
  if (req.method === "OPTIONS") {
    return new Response("ok", {
      headers: {
        "Access-Control-Allow-Origin": "*",
        "Access-Control-Allow-Methods": "POST, OPTIONS",
        "Access-Control-Allow-Headers": "Content-Type, Authorization",
      },
    });
  }

  try {
    const {
      fixed_blocks,    // array of { title, day_of_week, start_time, end_time }
      sleep_schedule,  // { sleep_time, wake_time }
      available_hours, // { monday: 5.0, tuesday: 3.0, ... }
      weekly_goals,    // array of { pillar_id, pillar_name, pillar_icon, goal_text }
      week_start,      // "YYYY-MM-DD" (Monday)
      week_dates,      // { monday: "2026-03-09", tuesday: "2026-03-10", ... }
    } = await req.json();

    const userMessage = `
Here is the user's information for the week starting ${week_start}:

FIXED SCHEDULE (non-negotiable commitments):
${JSON.stringify(fixed_blocks, null, 2)}

SLEEP SCHEDULE:
- Sleep at: ${sleep_schedule.sleep_time}
- Wake at: ${sleep_schedule.wake_time}

AVAILABLE FREE HOURS PER DAY (already accounts for fixed blocks, sleep, meals):
${Object.entries(available_hours).map(([day, hrs]) => `  ${day}: ${hrs} hours`).join("\n")}

WEEK DATES:
${Object.entries(week_dates).map(([day, date]) => `  ${day}: ${date}`).join("\n")}

WEEKLY GOALS BY PILLAR:
${weekly_goals.map((g: any) => `  ${g.pillar_icon} ${g.pillar_name} (pillar_id: ${g.pillar_id}): "${g.goal_text}"`).join("\n")}

Generate a complete weekly task schedule. Remember to use the exact pillar_id values provided and the exact date strings from WEEK DATES.`;

    const response = await fetch("https://api.anthropic.com/v1/messages", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "x-api-key": ANTHROPIC_API_KEY!,
        "anthropic-version": "2023-06-01",
      },
      body: JSON.stringify({
        model: "claude-sonnet-4-20250514",
        max_tokens: 4096,
        system: SYSTEM_PROMPT,
        messages: [{ role: "user", content: userMessage }],
      }),
    });

    const data = await response.json();
    const text = data.content?.[0]?.text || "";

    // Parse the JSON from Claude's response
    // Strip markdown code fences if present
    const cleaned = text.replace(/```json\n?/g, "").replace(/```\n?/g, "").trim();
    const result = JSON.parse(cleaned);

    return new Response(JSON.stringify(result), {
      headers: {
        "Content-Type": "application/json",
        "Access-Control-Allow-Origin": "*",
      },
    });
  } catch (error) {
    return new Response(JSON.stringify({ error: error.message }), {
      status: 500,
      headers: {
        "Content-Type": "application/json",
        "Access-Control-Allow-Origin": "*",
      },
    });
  }
});
```

Set the `ANTHROPIC_API_KEY` as a Supabase secret:
```bash
supabase secrets set ANTHROPIC_API_KEY=sk-ant-...
```

**Available hours calculation logic:**
```javascript
function calculateAvailableHours(dayOfWeek, fixedBlocks, sleepSchedule) {
  const TOTAL_DAY_MINUTES = 24 * 60;
  
  // Calculate sleep minutes
  let sleepMinutes;
  const sleepTime = parseTime(sleepSchedule.sleep_time); // minutes from midnight
  const wakeTime = parseTime(sleepSchedule.wake_time);
  if (sleepTime > wakeTime) {
    // Crosses midnight (e.g., 23:00 - 07:00)
    sleepMinutes = (TOTAL_DAY_MINUTES - sleepTime) + wakeTime;
  } else {
    sleepMinutes = wakeTime - sleepTime;
  }
  
  // Get fixed blocks for this day
  const dayBlocks = fixedBlocks.filter(b => b.day_of_week === dayOfWeek && b.is_active);
  const fixedMinutes = dayBlocks.reduce((sum, block) => {
    return sum + (parseTime(block.end_time) - parseTime(block.start_time));
  }, 0);
  
  // Meals + self-care: estimate 90 min/day (breakfast 20, lunch 30, dinner 30, misc 10)
  const mealMinutes = 90;
  
  const availableMinutes = TOTAL_DAY_MINUTES - sleepMinutes - fixedMinutes - mealMinutes;
  return Math.max(0, availableMinutes / 60); // convert to hours, floor at 0
}
```

```javascript
function getNextTask(todayTasks, currentTime) {
  // 1. If a task is "in_progress", always show that one
  const inProgress = todayTasks.find(t => t.status === 'in_progress');
  if (inProgress) return inProgress;
  
  // 2. Otherwise, find the first "pending" task (by sort_order)
  const pending = todayTasks
    .filter(t => t.status === 'pending')
    .sort((a, b) => a.sort_order - b.sort_order);
  
  if (pending.length > 0) return pending[0];
  
  // 3. If no pending tasks, all are done/skipped
  return null;
}
```

```javascript
function calculateWeekScore(tasks) {
  const scorableTasks = tasks.filter(t => 
    t.status !== 'skipped_emergency' // don't penalize emergencies
  );
  
  if (scorableTasks.length === 0) return 0;
  
  const completed = scorableTasks.filter(t => t.status === 'completed').length;
  const partial = scorableTasks.filter(t => t.status === 'partial').length;
  
  const score = ((completed + partial * 0.5) / scorableTasks.length) * 10;
  return Math.round(score * 10) / 10; // one decimal
}

function calculatePillarScore(tasks, pillarId) {
  const pillarTasks = tasks.filter(t => t.pillar_id === pillarId);
  return calculateWeekScore(pillarTasks);
}
```

### `lib/dates.js`

```javascript
import { startOfWeek, endOfWeek, addDays, format, isToday, 
         isBefore, isAfter, parseISO } from 'date-fns';

// Get Monday of current week (ISO standard)
export function getWeekStart(date = new Date()) {
  return startOfWeek(date, { weekStartsOn: 1 }); // 1 = Monday
}

// Get all 7 dates of a week as an object
export function getWeekDates(weekStart) {
  const days = ['monday', 'tuesday', 'wednesday', 'thursday', 'friday', 'saturday', 'sunday'];
  const dates = {};
  days.forEach((day, i) => {
    dates[day] = format(addDays(weekStart, i), 'yyyy-MM-dd');
  });
  return dates;
}

// Get ISO day of week (1=Mon, 7=Sun)
export function getISODayOfWeek(date) {
  const d = new Date(date).getDay();
  return d === 0 ? 7 : d;
}

// Format time for display: "14:00" → "2:00 PM"
export function formatTime(timeStr) {
  const [h, m] = timeStr.split(':').map(Number);
  const ampm = h >= 12 ? 'PM' : 'AM';
  const hour = h % 12 || 12;
  return `${hour}:${m.toString().padStart(2, '0')} ${ampm}`;
}
```

```jsx
import { createContext, useContext, useEffect, useState } from 'react';
import { supabase } from '../lib/supabase';

const AuthContext = createContext({});

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);
  const [profile, setProfile] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // Get initial session
    supabase.auth.getSession().then(({ data: { session } }) => {
      setUser(session?.user ?? null);
      if (session?.user) fetchProfile(session.user.id);
      else setLoading(false);
    });

    // Listen for auth changes
    const { data: { subscription } } = supabase.auth.onAuthStateChange(
      async (event, session) => {
        setUser(session?.user ?? null);
        if (session?.user) await fetchProfile(session.user.id);
        else { setProfile(null); setLoading(false); }
      }
    );

    return () => subscription.unsubscribe();
  }, []);

  async function fetchProfile(userId) {
    const { data } = await supabase
      .from('profiles')
      .select('*')
      .eq('id', userId)
      .single();
    setProfile(data);
    setLoading(false);
  }

  async function updateProfile(updates) {
    const { data } = await supabase
      .from('profiles')
      .update(updates)
      .eq('id', user.id)
      .select()
      .single();
    setProfile(data);
    return data;
  }

  return (
    <AuthContext.Provider value={{ user, profile, loading, updateProfile, fetchProfile }}>
      {children}
    </AuthContext.Provider>
  );
}

export const useAuth = () => useContext(AuthContext);
```

```javascript
// In the component that triggers AI generation (GoalsStep or WeekPlanPage):

async function generateWeeklyTasks(weekStart) {
  // 1. Fetch all needed data
  const { data: fixedBlocks } = await supabase
    .from('fixed_blocks').select('*').eq('user_id', user.id);
    
  const { data: sleepData } = await supabase
    .from('sleep_schedule').select('*').eq('user_id', user.id).single();
    
  const { data: pillars } = await supabase
    .from('pillars').select('*').eq('user_id', user.id).order('sort_order');
    
  const { data: goals } = await supabase
    .from('weekly_goals').select('*').eq('user_id', user.id)
    .eq('week_start', format(weekStart, 'yyyy-MM-dd'));

  // 2. Calculate available hours per day
  const weekDates = getWeekDates(weekStart);
  const availableHours = {};
  const dayNames = ['monday','tuesday','wednesday','thursday','friday','saturday','sunday'];
  dayNames.forEach((day, i) => {
    const dayNum = i + 1; // 1=Mon, 7=Sun
    availableHours[day] = calculateAvailableHours(dayNum, fixedBlocks, sleepData);
  });

  // 3. Build weekly_goals payload with pillar info
  const weeklyGoals = goals.map(g => {
    const pillar = pillars.find(p => p.id === g.pillar_id);
    return {
      pillar_id: g.pillar_id,
      pillar_name: pillar.name,
      pillar_icon: pillar.icon,
      goal_text: g.goal_text,
    };
  });

  // 4. Call Edge Function
  const response = await supabase.functions.invoke('generate-tasks', {
    body: {
      fixed_blocks: fixedBlocks,
      sleep_schedule: sleepData,
      available_hours: availableHours,
      weekly_goals: weeklyGoals,
      week_start: format(weekStart, 'yyyy-MM-dd'),
      week_dates: weekDates,
    }
  });

  const { tasks: aiTasks, warnings, summary } = response.data;

  // 5. Insert tasks into DB
  const tasksToInsert = aiTasks.map(t => ({
    user_id: user.id,
    pillar_id: t.pillar_id,
    week_start: format(weekStart, 'yyyy-MM-dd'),
    day: t.day,
    scheduled_time: t.scheduled_time,
    title: t.title,
    completion_criteria: t.completion_criteria,
    duration_minutes: t.duration_minutes,
    sort_order: t.sort_order,
    energy_level: t.energy_level,
    source: 'ai_generated',
    status: 'pending',
  }));

  await supabase.from('tasks').insert(tasksToInsert);

  return { warnings, summary };
}
```

## CRITICAL UX RULES

1. **The dashboard must load in under 2 seconds.** Fetch only today's tasks, not the whole week.
2. **The "Next Task" card must be the FIRST thing a user sees** when opening the app. Not a list. Not a calendar. One task.
3. **Never show a blank state without guidance.** Every empty state has a message and a CTA.
4. **The check-in (Start → Done) flow must work in under 5 taps.** Open app → see task → tap Start → tap Done. That's it.
5. **AI generation should feel fast.** Show the animated loading screen. If it takes over 10 seconds, show a "Still working..." message. If over 30 seconds, show a retry button.
6. **All time displays use 12-hour format** (2:00 PM, not 14:00) for user-facing UI. Database stores 24-hour format.
7. **Mobile is the primary experience.** Design at 375px width first. Desktop is a nice-to-have.
8. **Dark mode only for MVP.** Background: slate-900. Cards: slate-800. Text: slate-50/slate-400.
9. **Every button press must have immediate visual feedback** — loading spinner, color change, or animation.
10. **Data auto-saves.** No "save" buttons on forms. Use debounced writes or save on blur/next.
